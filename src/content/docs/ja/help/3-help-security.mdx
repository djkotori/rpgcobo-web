RPG-Cobo は、ユーザーの皆さんが安心してゲームを作れるように、**特にプラグインの仕組みを安全に利用できるように**、強力なセキュリティ機能が組み込まれています。

一般的なプラグインの仕組みを持つツール(*例えば Unity, Blender, Visual Studio Code など*)は、マルウェアなど悪意のあるコードの実行を防ぐ有効な手段がなく、
プラグインをインストールする際に、**「このプラグインは安全か？」といった判断はユーザー自身に委ねられてしまっています。**

その点、RPG-Cobo のプラグインは非常にセキュアに組み込むことができるように設計されています。  
このページでは、RPG-Cobo のセキュリティの仕組みと、セキュリティの制限に関する設定方法について解説します。

### <M code="e65f"/> 悪意あるプラグインが起こすセキュリティリスク
---

以下に書かれているようなことは、悪意のあるプラグインがユーザーのPC上で実行されてしまった場合に起こり得るリスクです。  
機密漏洩、データ破損、PCの乗っ取りなど、ユーザーにとって非常に危険なことが、セキュアでないプラグインでは起こり得ます。

1. 悪意のある外部プロセスを実行されてしまう。
2. 任意のネットワーク通信。PC内の情報や、ユーザーの個人情報を外部に送信されてしまう。
3. 任意のファイルの書き出し・削除。実行ファイルを書き換えられて攻撃コードを仕込まれたり、ユーザーの大事なファイルの削除や暗号化をされてしまう。


### <M code="e65f"/> なぜセキュリティリスクは起こるのか
---

悪意のあるプラグインがセキュリティリスクを引き起こす主な原因は、プラグインがOSのリソースに自由にアクセスできることにあります。  
既存ツールのプラグインシステムでは、プラグインがアクセスできるリソースに制限がなく、ユーザーのPCに対して危険な操作を行うことが可能です。

例えば、 Blender は、プラグインの実行を Pythonスクリプト で行いますが、Pythonの基本システムである`os` `subprocess` `socket`等のモジュールはユーザーのPC上のリソースへのアクセスを制限できません。
悪意のあるプラグインがユーザーのPC上で実行されてしまった場合、ユーザーのPCに対する危険な操作を防ぐ手段が Blender 側には無いのです。

同じように、Unity は .NET、 Visual Studio Code は Node.js の仕組みでプラグインを実行するため、リソースへのアクセスを制限できません。


### <M code="e65f"/> RPG-Coboのセキュリティの仕組み
---

RPG-Cobo は、**ツールからアクセスする各種リソース(ファイル、ネットワーク、コード実行など)に対してネイティブレベルの厳格なアクセス制限を設けています。**

デフォルト設定では、プラグインを含むすべてのコードは以下のような制限を受けます。

- 外部コードの実行制限
  - `/$ProjectDir/rpg-cobo.exe`, `/$ProjectDir/tools`以下の実行ファイル 以外のコードは実行できません。
- ファイル入出力の制限
  - 拡張子が`exe` `dll` `bat` `ps1` `psm1` のファイルは書き込みできません。
  - `/$ProjectDir`以下、共通のデータフォルダ 以外のフォルダへの書き込みはできません。
  - すべてのファイルの読み込みは可能です。
- ネットワーク通信の制限(IP)
  - ローカルホスト(127.0.0.1)に対してのみTCP/UDPへの接続が可能です。
  - ローカルホスト又はプライベートIP(127.0.0.1, 192.168.x.x, 172.16.x.x)に対してのみTCP/UDPサーバーのbindが可能です。
- ネットワーク通信の制限(https)
  - 下記のサーバー名に対してのみHTTPS通信が可能です。  
    `rpg-cobo.com` `*.rpg-cobo.com` `*.googleapis.com` `translate.google.com`
- ウェブのブラウザ表示の制限
  - 下記のURLに対してのみ、ウェブブラウザを表示させることが可能です。  
    `https://rpg-cobo.com/**` `https://*.rpg-cobo.com/**` `https://github.com/djkotori/**` `http://localhost:*/**` `http://192.168.*.*:*/**` `http://172.16.*.*:*/**` `file://$cdir/**/*.html` `dir://**`
- ネイティブアドオンの制限
  - RPG-Cobo ツールが提供するネイティブアドオンのみが使用可能です。  
    ***ネイティブアドオンは、RPG-Coboのネイティブコードを拡張します。ユーザーは基本的にネイティブアドオンを作ることはできません。***

これらのアクセス制限により、悪意のあるプラグインがユーザーのPC上で実行されてしまった場合でも、ユーザーのPCに対して危険な操作を行うことはできません。

### <M code="e65f"/> セキュリティの制限を緩和する方法
---

RPG-Cobo は堅固なセキュリティを備えていますが、本格的なゲームを開発する際には、この制限を緩和したいと考えることはあると思います。  
例えば、ネットワーク通信を行うプラグインを作りたい場合や、ユーザーのPC内のファイルにアクセスするプラグインを作りたい場合などです。

セキュリティ上のリスクがあることを理解した上で、RPG-Coboのセキュリティの制限を緩和する方法は以下の通りです。

<Steps>
1. アクティビティバーの「<M code="e8b8"/>」から「<M code="e8b8"/> 環境設定 (Ctrl+,)」を開きます。
2. 「ツールのセキュリティ」の項目で、デフォルト設定では「 **Lv2:[推奨]ネット、ファイルのセキュリティが有効** 」となっています。  
    
    <SS src='docs/help-security-1.png' alt=''/>  
    このセキュリティレベルは3段階に分かれており、以下のようになっています。  
    - Lv0: アクセス制限を全解除 : ***危険！全てのアクセスが可能になります。***
    - Lv1: ネットのアクセス制限を解除 : 情報漏洩のリスクがあります。
    - Lv2: ネット、ファイルのセキュリティが有効 (推奨)
3. 設定の変更時に、確認のダイアログが出ます。  
    ここで「はい」を選択すると、セキュリティの制限が緩和されます。  
    なお、セキュリティの制限を緩和する場合は、**必ず信頼できるプラグインのみを使用するようにしてください。**
</Steps>
