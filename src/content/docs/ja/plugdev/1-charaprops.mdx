「**プラグイン開発実践編**」の章では、サンプルプラグインを使って、RPG-Coboのプラグイン開発の実践的な方法を解説していきます。

まずは、キャラの特徴を表すプロパティを追加するためのプラグインを作成してみましょう。

### このページで解説するプラグインについて
---
下記のサンプルプラグインを解説します。

- work/plugin-samples/charaprops.plugsk : キャラのパラメータを追加します。プラグインのプロパティも定義します。

<Aside>
`work/plugin-samples/charaprops.plugsk` が見当たらないという方は、ポータルアプリを最新の状態にしてからで RPG-Cobo のプロジェクトを作成してください。  
少し古い状態のプロジェクトには、サンプルプラグインが入っていない場合があります (2026/07/13 時点)。
</Aside>

### charaprops.plugsk の概要
---
このプラグインは、以下の2つのキャラクターのパラメータを追加します。
1. 幸運スコアプロパティ(LUK) : プレイヤーとエネミーの両方に追加される**スコアプロパティ**です。
2. 満腹度ステータスプロパティ(FOOD) : プレイヤーのみに追加される**ステータスプロパティ**です。

まず、**スコアプロパティ**というのは、キャラが持つ数値のパラメータで、レベルに応じて変化したり、装備やオプションで変化することができるパラメータです。
例えば、物理攻撃力(PATK)や魔法攻撃力(MATK)などがスコアプロパティに該当します。

一方、**ステータスプロパティ**というのは、キャラが持つ数値のパラメータで、ゲームの色々な状況に応じて変数的に変化するパラメータです。
例えば、HPやMP、装備、ジョブなどがステータスプロパティに該当します。

`charaprops.plugsk`はその2つとも追加しています。さらに、プラグインのパラメータとして、どちらを追加するかを指定できるようにしています。


それではまずは、サンプルプラグインをそのまま見てみましょう(コメントは一部省略しています)。

```js
import PropertyType;

local luklbl = { "ja":"幸運","en":"Luck"};
local foodlbl = { "ja":"満腹度","en":"Satiety"};

pluginfo <- {
	name : { "ja":"サンプルプロパティ追加","en":"Sample Properties Addition"}
	author : "djkotori"
	ver : "0.0.1"
	desc : { "ja":"キャラにサンプルプロパティを追加します。","en":"Adds sample properties to chara."}
	//	プラグインのプロパティを定義する。プラグインマネージャーで設定可能。
	props : [
		{ pid : "LUK", lbl : luklbl, type : BOOL, defval : true },		//	幸運スコアプロパティを追加するかどうか
		{ pid : "FOOD", lbl : foodlbl, type : BOOL, defval : true },	//	満腹度ステータスプロパティを追加するかどうか
	]
	order : 50	//	読み込み順。>=1 なら何でも良し
}

function loadPlugin( args, opt){
	//	LUK定義。プラグインでLUKがtrueなら追加する
	if( opt["LUK"]){
		local prop = {
			proptype = "score"
			pid = "LUK"
			lbl = luklbl
			lvcurve = true		//	レベルに応じて変化するパラメータ。
			//	プロパティ定義。lvcurveの編集でも参照
			type = SCALAR
			range = (1,9999)
			defval = 100
		};
		//	プレイヤー、エネミーのプロパティに追加。※バトルでLUKを使用するためには必ずプレイヤー、エネミーの両方に追加する必要がある。
		RPGSystem.playerprops .. prop;
		RPGSystem.enemyprops .. prop;
		//	ジョブ画面、レベルアップ時の表示に追加。装備画面ではLUKを表示させない
		//::equipmenu_showpids .. "LUK";
		::jobmenu_showpids .. "LUK";
		::levelup_showpids .. "LUK";
	}
	//	FOOD定義。プラグインでFOODがtrueなら追加する
	//	満腹度はキャラ固有のステータス値で、時間で減っていき0になるとHPが減るような仕様にすることもできる。
	if( opt["FOOD"]){
		local prop = {
			proptype = "status"
			pid = "FOOD"
			lbl = foodlbl
			type = SCALAR
			range = (1,100)
			defval = 100
		};
		//	満腹度はプレイヤーのみのプロパティ
		RPGSystem.playerprops .. prop;
		RPGPlayer.addStatusProp( "FOOD", 100);
		//	満足度オプションをプレイヤーにつける。walkで減る、0で空腹状態を付けてHPを減らすなどの処理を追加できる。
		RPGSystem.option .. {
			category = -1,		//	カテゴリ-1で追加不能に。
			key = "opt_food",
			name = foodlbl,
			icon = "\ue000",
			order = 99,
			desc = "",
			//	オプションの効果実行
			action = @( opd, id, arg, self){
				switch( id){
				case "walk":	//	1m歩行ごとに呼ばれる。
					self.FOOD = Math.max( self.FOOD-0.1, 0);	//	歩行距離の10%減少。1kmで100減少する。
					break;
				}
			}
		}
		//	プレイヤーのgetBaseOptionsをフックして、満腹度オプションを追加する。※スパゲッティ化するのでより良い方法を検討！！
		local opd = { opttype = "opt_food", };
		local _getBaseOptions = RPGPlayer.getBaseOptions;
		RPGPlayer.getBaseOptions <- @(){		//	インスタンス関数を入れ替える。"<-"演算子を使う必要あり
			local opts = _getBaseOptions();		//	元々のgetBaseOptionsを呼ぶ
			opts .. RPGOption( opd);			//	満腹度オプションを追加する。これで全てのプレイヤーが満腹度オプションを持つことになる。
			return opts;
		}
	}
	return true;
}
```

…最初から中々に長いコードになってしまいましたが、中身を分解すれば一つ一つはとても単純です。  
これより順を追って説明していきます。


### 1: 幸運プロパティ(LUK)の追加
---

LUKを追加する部分を抜き出すと以下のようになります。

```js
function loadPlugin( args, opt){
	//	LUK定義。プラグインでLUKがtrueなら追加する
	local prop = {
		proptype = "score"
		pid = "LUK"
		lbl = { "ja":"幸運","en":"Luck"}
		lvcurve = true		//	レベルに応じて変化するパラメータ。
		//	プロパティ定義。lvcurveの編集でも参照
		type = SCALAR
		range = (1,9999)
		defval = 100
	};
	//	プレイヤー、エネミーのプロパティに追加。※バトルでLUKを使用するためには必ずプレイヤー、エネミーの両方に追加する必要がある。
	RPGSystem.playerprops .. prop;
	RPGSystem.enemyprops .. prop;
	//	ジョブ画面、レベルアップ時の表示に追加。装備画面ではLUKを表示させない
	//::equipmenu_showpids .. "LUK";
	::jobmenu_showpids .. "LUK";
	::levelup_showpids .. "LUK";
}
```

- `local prop = { ... }`で、幸運プロパティの定義を作成しています。
  - `proptype="score"` : スコアプロパティとして定義することを示します。
  - `pid="LUK"` : プロパティIDを"LUK"に設定します
  - `lvcurve=true` : レベルに応じて変化するパラメータであることを示します。falseや省略で、レベルで変化しないパラメータになります。
  - 値の定義としては、`type=SCALAR`, `range=(1,9999)`, `defval=100`としています。これは、1～9999の範囲の数値で、デフォルト値は100であることを示します。
- `RPGSystem.playerprops .. prop;`で、プレイヤーのプロパティリストに追加します。  
- `RPGSystem.enemyprops .. prop;`で、エネミーのプロパティリストにも追加します。  
   ***⇒ バトルでLUKを使用するためには、プレイヤーとエネミーの両方に追加する必要があります。***
- `::jobmenu_showpids .. "LUK";`で、ジョブ画面のパラメータ表示にLUKを表示するように設定します。
- `::levelup_showpids .. "LUK";`で、レベルアップ時の表示にLUKを追加します。lvcurve=trueのプロパティは、レベルアップ時に変化する値を表示するために、レベルアップ画面に表示したほうが良いでしょう。

さあ、これで、幸運プロパティ(LUK)の追加は完了です。しかし、これだけだと幸運はただの数字でしかありません。  
実際に幸運が上がることで計算式を変化させるには、タイプ設定の「**ダメージ式**」でLUKを計算式に入れる必要があります。


### 2: 満腹度プロパティ(FOOD)の追加
---

次は、満腹度プロパティ(FOOD)の追加です。こちらは、プレイヤーのみに追加されるステータスプロパティです。
少し趣向を凝らして、歩くと減っていくパラメータにします。満腹度が0になると、HPが減るような仕様にすることもできます。
満腹度が減る仕組みは、オプションのタイプを追加することで実装されています。

```js
function loadPlugin( args, opt){
	local prop = {
		proptype = "status"
		pid = "FOOD"
		lbl = { "ja":"満腹度","en":"Satiety"}
		type = SCALAR
		range = (1,100)
		defval = 100
	};
	//	満腹度はプレイヤーのみのプロパティ
	RPGSystem.playerprops .. prop;
	RPGPlayer.addStatusProp( "FOOD", 100);
	//	満足度オプションをプレイヤーにつける。walkで減る、0で空腹状態を付けてHPを減らすなどの処理を追加できる。
	RPGSystem.option .. {
		category = -1,		//	カテゴリ-1で追加不能に。
		key = "opt_food",
		icon = "\ue000",
		order = 99,
		//	オプションの効果実行
		action = @( opd, id, arg, self){
			switch( id){
			case "walk":	//	1m歩行ごとに呼ばれる。
				self.FOOD = Math.max( self.FOOD-0.1, 0);	//	歩行距離の10%減少。1kmで100減少する。
				break;
			}
		}
	}
	//	プレイヤーのgetBaseOptionsをフックして、満腹度オプションを追加する。※スパゲッティ化するのでより良い方法を検討！！
	local opd = { opttype = "opt_food", };
	local _getBaseOptions = RPGPlayer.getBaseOptions;
	RPGPlayer.getBaseOptions <- @(){
		local opts = _getBaseOptions();
		opts .. RPGOption( opd);
		return opts;
	}
}
```

- `local prop = { ... }`の仕組みは`LUK`とほぼ同じです。`proptype = "status"`として、ステータスプロパティとして定義しています。
- `RPGPlayer.addStatusProp( "FOOD", 100);`で、RPGPlayerクラスに追加の変数として`FOOD`を追加しています。  
   これにより、RPGPlayer.FOODを参照、変更、保存することができます。
- `RPGSystem.option .. { ... }`で、満腹度低下オプションを追加しています。  
   このオプションは、プレイヤーの歩行距離に応じてFOODを減少させる処理を実装しています。
- `RPGPlayer.getBaseOptions <- @(){ ... }`で、関数を上書きして全プレイヤーがオプションに満腹度消費オプションを追加しています。

#### 改造案

この`FOOD`プロパティは、このままだと何も起こりません。満腹度が0になったときにHPを減らすなどの処理を追加することで、ゲーム性を高めることができます。  
例えばオプションの`action`関数を改造して、満腹度が0になったときに「空腹コンディション」にする処理を追加することができます。
また、食料をアイテムとして持っていると、自動的に消費して満腹度を回復するような処理を追加するのも面白いでしょう。


### 3: プラグインのプロパティについて

このプラグインは、プラグインマネージャーで設定可能なプロパティを定義しています。

```js
pluginfo <- {
	...,
	props : [
		{ pid : "LUK", lbl : luklbl, type : BOOL, defval : true },		//	幸運スコアプロパティを追加するかどうか
		{ pid : "FOOD", lbl : foodlbl, type : BOOL, defval : true },	//	満腹度ステータスプロパティを追加するかどうか
	]
}
```

この指定をすることで、プラグインマネージャーでは以下の画像のようにプロパティの設定項目が表示されます。
<SS src='docs/charaprops-1.png' alt=''/>

プラグインマネージャーで設定した値は、`loadPlugin( args, opt)`関数の`opt`引数にテーブルとして渡されます。
以下のようにすると、設定値を元にプラグインのロード処理を変えることが出来るわけです。

```js
function loadPlugin( args, opt){
	if( opt["LUK"]){
		//	LUKを追加する処理
	}
	if( opt["FOOD"]){
		//	FOODを追加する処理
	}
	return true;
}```
