RPG-Cobo はオープンソースで提供されており、ゲームの動作やツールの拡張などコードを自由に改変することができます。
また、全てのコードは`sk`スクリプトで記述されているため、ビルドの必要がなく、ソースコードを直接編集すればすぐに動作を確認することができます。

エンジン内部のソースを直接編集する方法は一番手軽で直接的ですが、問題点も多くあまりお勧めできません。例えば、RPG-Coboのバージョンアップにより変更されたソースファイルが
書き換えられていた場合、gitのマージが確実に機能するとは言えず、最悪の場合はゲームやツールが起動しなくなってしまいます。

## プラグインで機能拡張をしよう

プラグインを実装すればゲームやツールのソースコードを直接変更することなく安全に機能を拡張できます。  
また、他の人の作ったプラグインをいくつも追加していき、ゲームを効率的に拡張していけるメリットも大きいです。

||ソース改変|プラグイン|
|-|---|---|
|実装の手軽さ|◎|〇|
|バージョンアップ対応|×|〇|
|拡張性|×|〇|

## プラグインの管理方法

ファイルメニュー⇒プラグインマネージャーを選択するとプラグインマネージャーのダイアログが表示されます。

1. 各プロジェクトの中でプラグインは `/$ProjectDir/project/plugin` に配置されています。
2. サンプルのプラグインが `/$ProjectDir/work/plugin-samples` に入っています。中にあるhello.pluginskをコピーしてプラグインフォルダに追加してみましょう。  
3. 追加後に「ツールを再ロード(Ctrl+F4)」することで、追加したプラグインがプラグインマネージャーで表示されます。

それでは、追加されたプラグインをリストで選択し、右上のプルダウンメニューから有効化しましょう。

有効／無効の他にも、プラグイン毎に指定するプロパティを設定することができます。

プラグインの設定を変更した場合に、ツールはリロードされ設定が反映されます。


## 最も簡単なプラグイン

プラグインはファイル一つで作ることができます。もっとも簡単なプラグインを以下に紹介します。  
*ファイルは必ずutf-8で保存しましょう。*

```JavaScript
//hello.plugsk

pluginfo <- {
	name : { "ja":"こんにちはプラグイン","en":"Hello Plugin"}
	auther : "djkotori"
	ver : "0.0.1"
	desc : { "ja":"最も簡単なプラグインの例です。","en":"The simplest plugin example."}
	props : [
		{ pid:"message", lbl:"Hello message", type:PropertyType.STRING, defval="RPG-Cobo" }
	]
}

function loadPlugin( args, opt){
	printf( "--------------- Hello %S! ---------------\n", opt.message);
	return true;
}
```

このプラグインは機能といえるようなものは何もなく、ただ単にプラグインのロード時に文字列をコンソールに出力するだけのものです。  
ですが、RPG-Coboでプラグインを実装するために必要十分な構造を表しています。

- pluginfoテーブル : 
	プラグインの登録情報をテーブルで埋め込みます。このプラグインではname,auther,ver,desc,propsのプロパティを定義していますが、より多くのプロパティが存在します。
	詳しくは、リファレンス：プラグイン構造の詳細を参照ください。
- loadPlugin関数 : 
	有効化されたプラグインの読み込みに実行される関数です。この関数内で既存の処理の動作を変更したり、定義を追加したり、フックを追加してプログラムの動作を拡張します。

*※pluginfo内のpropsプロパティは、必要ない場合は定義しなくても構いませんが、定義することでプラグインマネージャーのUIからプラグインの動作に影響を与えるプロパティを設定できるようになります。*
