### <M code="f10d"/> 2つのプラグイン形式
---
RPG-Cobo では、プラグインの形式として以下の2つの形式をサポートしています。

#### 1. 単一のプラグインスクリプト形式（plugsk形式）
  - プラグインフォルダに単一のファイル( ***plugin名.plugsk*** )を配置するだけでプラグインとして認識されます。  
    テキストファイル一つで作れるのでとても手軽です。
  
#### 2. 複数のスクリプト・リソースをまとめたプラグインフォルダ形式(フォルダ形式)
  - プラグインフォルダ内に、フォルダを配置し、その中に ***plugin.sk*** を配置することでプラグインとして認識されます。  
    プログラムの拡張だけでなく、リソースやデータをまとめてプラグインとして配布することができます。



### <M code="f10d"/> プラグインスクリプトの定義のしかた
---
***plugin名.plugsk*** または ***/plugin名/plugin.sk*** は、特定の構造をした`sk`スクリプトである必要があります。  
スクリプトには、以下のようにプロパティと関数を定義しましょう。

```javascript
1:	pluginfo <- {
2:		name : { "ja":"プラグイン名","en":"Plugin Name"}
3:		auther : "djkotori"
4:		ver : "0.0.1"
5:		desc : { "ja":"最も簡単なプラグインの例です。","en":"The simplest plugin example."}
6:		order : 10
7:		props : [
8:			{ pid:"message", lbl:"Hello message", type:PropertyType.STRING, defval="RPG-Cobo" }
9:		]
10:	}
11:
12:	function loadPlugin( args, opt){
13:		return true;
14:	}
```

以下、プロパティと関数の定義について説明します。(必須)以外のプロパティは省略可能です。

##### `1-10: pluginfo` (必須)
このプラグインの登録情報を定義します。pluginfoはテーブルで定義され、続いて説明するプラグイン名、作者、バージョン、説明、プラグインマネージャーから設定可能なプロパティなどを定義します。

##### `2: name` (必須)
プラグインマネージャー上で表示されるプラグイン名です。string型か、テーブル型にして言語ごとの文字列を定義することができます。
<Aside>
RPG-Coboの文字列情報の多くは、テーブルを指定することでローカライズされた文字列を定義することができます。
</Aside>
##### `3: auther`
プラグインの作者名です。
##### `4: ver`
プラグインのバージョンです。現在のところ表示専用です。
##### `5: desc`
プラグインの説明です。プラグインの分かりやすい説明や使い方を書きましょう。
##### `6: order`
プラグインの読み込みの優先度を指定します。数値が小さいほど優先度が高く、先に読み込まれます。  
**RPG-Coboの標準プラグイン`rpgtools`は`order:0`です。** `rppgtools`よりも先に読み込む必要がある場合は、負の値を指定することもできますが、通常は正の値を指定することを推奨します。  
省略した場合は`order:100`となります。

##### `7-9: props`
プラグインマネージャーから設定可能なプロパティを定義します。propsは配列で定義され、各要素はテーブルでプロパティの情報を定義します。
<Aside>
プロパティ定義は、RPG-Coboの全てのデータ定義に使われている、RPG-Coboのデータ構造の根幹をなす重要な仕組みです。  
まずはRPG-Coboのソースコード内でどのように使われているかを確認してみましょう。
</Aside>
##### `8: props[0]`
プロパティ定義の一つを例として示します。
- `pid` : プロパティのIDです。プラグイン内でこのプロパティを参照する際に使用します。
- `lbl` : プロパティのラベルです。プラグインマネージャー上で表示される文字列です。
- `type` : プロパティの型です。PropertyType型で指定します。詳しくはリファレンスを参照してください。  
- `defval` : プロパティのデフォルト値です。プラグインマネージャー上でプロパティが未設定の場合に使用されます。

##### `12-14: function loadPlugin( args, opt)` (必須)
有効化されたプラグインの読み込みに実行される関数です。この関数内で既存の処理の動作を変更したり、定義を追加したり、フックを追加してプログラムの動作を拡張します。
プラグインのロード処理が正常に完了した場合は`true`を返し、失敗した場合は`false`を返します。Exceptionが発生した場合もプラグインのロードは失敗します。

- `args` : ゲームの起動時に指定されたコマンドライン引数が格納された配列です。通常は使用しません。
- `opt` : pluginfoで定義されたプロパティ定義が格納されたテーブルです。プラグインマネージャーで設定されたプロパティの値を参照することができます。

### <M code="f10d"/> 作成例：メニューを追加する
---

```javascript
	//	pluginfoは省略
1:	function loadPlugin( args, opt) {
2:		::skstudio.registerMenu( {
3:			uid = "menu_edit_pluginx"
4:			lbl = "PluginX"
5:			parentid = "menu_edit"
6:			icon = "\ue166"
7:			defkey = "^_L"
8:			where = null
9:			func = @( m){ printf( "PluginX menu selected!\n"); }
10:		});
11:		return true;
12:	}
```
このサンプルは、RPG-Coboの「編集」メニューに「PluginX」というメニューを追加するプラグインです。  
以下の情報を含むテーブルを`registerMenu`関数に渡すことで、メニューを追加することができます。

- `3: uid`(必須) : メニューのIDです。重ならないようなIDを指定してください。
- `4: lbl`(必須) : メニューに表示される文字列です。
- `5: parentid` : 親メニューのIDです。ここでは「編集」メニューのIDを指定しています。
- `6: icon` : メニューに表示されるアイコンです。RPG-Coboのアイコンフォントのコードポイントを指定します。
- `7: defkey` : メニューのショートカットキーです。RPG-Coboのショートカットキーの定義形式で指定します。
- `8: where` : メニューの表示条件を指定します。
- `9: func` : メニューが選択されたときに実行される関数です。引数としてメニュー情報が渡されます。

※`submenu`を指定することで、サブメニューを追加することもできます。

### <M code="f10d"/> プラグイン処理を書くときの注意
---
**システム全体に副作用のある処理は、`loadPlugin`関数内で実行されるようにする必要があります。絶対に関数外で実行しないでください。**  
例えば`loadPlugin`関数外でメニューを追加する処理を入れた場合、プラグインが無効化されている状態でもメニューが追加されてしまい、プラグインの有効化・無効化の意味がなくなってしまいます。
また、プラグイン実行順の規則も破壊されてしまいます。

特に、フォルダ形式のプラグインで他のスクリプトやリソースを読み込む場合は、`loadPlugin`関数内で読み込むように気を付けましょう。

```javascript
	pluginfo <- { ... };

	::skstudio.registerMenu( { ... });	//	このような書き方はNG！

	function loadPlugin( args, opt) {
		return true;
	}
```
